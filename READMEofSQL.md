# 🎵 Music Streaming Database (SQL Project)

## 📌 Overview
This project focuses on performing **SQL operations** on a **Music Streaming Database** using **MySQL Workbench**. The study involves creating and managing structured tables, executing queries for data retrieval, and ensuring **normalization, referential integrity, and performance optimization**. The database is designed to handle **users, playlists, tracks, albums, artists, music labels, and contracts**, facilitating efficient music data management.

## 📄 Project Report
The full report, including SQL operations, query execution, and database analysis, can be accessed from the repository. ("SQL Project Report" file)

## 🛠️ Technologies Used
- **MySQL Workbench** - Database design and management
- **SQL Queries** - Data retrieval, modification, and analysis
- **Normalization Techniques (1NF, 2NF)** - Ensuring database efficiency
- **Stress Testing & Query Optimization** - Performance evaluation

## 📊 Key Features
- 📂 **Database Creation & Table Management**
- 🔄 **Normalization & Referential Integrity Checks**
- ✅ **Primary Key, Composite Key, and Foreign Key Constraints**
- 🔍 **Advanced SQL Queries for Managerial Insights**
- 📈 **Performance Optimization & Stress Testing**

## 📌 SQL Operations Performed
- **Data Insertion**: Populated tables with structured data.
- **Normalization Checks**: Ensured compliance with **1NF and 2NF**.
- **Foreign Key Validations**: Maintained database relationships.
- **Stress Testing**: Evaluated system performance under different operations.
- **Query Execution for Business Insights**, including:
  - **Top Artists Based on Track Count**
  - **Most Popular Playlists**
  - **Subscription Type Analysis**
  - **Music Label Performance**
